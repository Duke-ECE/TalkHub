# TalkHub 登录功能更新说明（第一阶段-步骤2）

本文档记录本次“用户登录”功能的落地实现，目标是让人类开发者与 AI 代理都能快速理解当前状态并继续迭代。

## 1. 目标与范围

对应需求：`第一阶段 -> 2. 用户登录`

已完成：

- 后端提供登录接口：`POST /api/auth/login`
- 登录成功返回 JWT 和用户基本信息
- 前端完成登录页、登录态持久化、退出登录
- 管理员账号由环境变量初始化
- 禁止自动注册（不存在用户时直接登录失败）

未包含：

- 完整注册流程
- 权限分级（仅实现最小登录能力）
- JWT 鉴权中间件（将在后续 WebSocket/接口鉴权阶段接入）

## 2. 后端更新

### 2.1 核心能力

- 登录服务 `AuthService`：
  - 根据用户名查询用户
  - 使用 `BCrypt` 校验密码
  - 校验通过后签发 JWT
- 登录接口 `AuthController`：
  - 路径：`/api/auth/login`
  - 方法：`POST`
  - 入参：`username` + `password`
  - 出参：`token` + `user`

### 2.2 数据模型

本次使用并落地了 `users` 表（`schema.sql`）：

- `id`
- `username`（唯一）
- `password_hash`
- `nickname`
- `avatar_url`
- `status`
- `created_at`
- `updated_at`

### 2.3 管理员初始化机制（新增）

应用启动时会执行 `AdminBootstrapRunner`：

- 从环境变量读取管理员账号密码
- 若管理员不存在则创建
- 若管理员存在则同步更新密码哈希（便于改密）

这使得系统启动后至少有一个可登录账号，无需注册流程。

### 2.4 JWT 生成

- 使用 `jjwt`（HS256）
- payload 包含：
  - `sub`（用户 ID）
  - `username`
  - `iat`
  - `exp`
- 签名密钥来自 `JWT_SECRET`
- `JWT_EXPIRES_SECONDS` 控制有效期

## 3. 前端更新

前端 `App.tsx` 已改为登录流程页面：

- 登录表单调用 `POST /api/auth/login`
- 登录成功将 `token + user` 存入 `localStorage`（键：`talkhub_auth`）
- 页面刷新后恢复登录态
- 提供退出登录按钮（清理本地登录态）

## 4. CORS 与本地联调

已新增 `CorsConfig`，放开：

- `http://localhost:*`
- `http://127.0.0.1:*`

允许前端开发端口动态变化时仍能访问后端 API。

## 5. 环境变量

除既有变量外，本次新增/重点使用：

- `JWT_SECRET`
- `JWT_EXPIRES_SECONDS`
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `ADMIN_NICKNAME`

示例（参考根目录 `.env.example`）：

```env
JWT_SECRET=change-me
JWT_EXPIRES_SECONDS=86400
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin123456
ADMIN_NICKNAME=Administrator
```

## 6. API 契约

### 6.1 登录请求

`POST /api/auth/login`

```json
{
  "username": "admin",
  "password": "admin123456"
}
```

### 6.2 登录成功响应（200）

```json
{
  "token": "jwt-token",
  "user": {
    "id": 1,
    "username": "admin",
    "nickname": "Administrator",
    "avatarUrl": null
  }
}
```

### 6.3 登录失败响应（401）

- 用户不存在
- 密码错误

## 7. 启动与验证

推荐命令：

```bash
make install
make run
```

验证方式：

1. 打开前端页面
2. 使用 `ADMIN_USERNAME / ADMIN_PASSWORD` 登录
3. 观察是否返回登录态并写入本地存储

## 8. 排障指南

### 8.1 “无法连接后端”

- 检查后端是否成功启动
- 检查页面上的 API 地址与后端端口是否一致
- 检查浏览器控制台是否有网络错误

### 8.2 登录 401

- 检查 `.env` 中 `ADMIN_USERNAME / ADMIN_PASSWORD` 是否与输入一致
- 修改管理员密码后需重启后端使初始化逻辑生效

### 8.3 数据库相关报错

- 检查 PostgreSQL 连接参数
- 检查账号是否有连接/建库权限（已启用建库初始化逻辑）

## 9. 给后续 AI 代理的约束

- 当前版本不允许自动注册，登录仅面向已存在用户
- 管理员账号由环境变量初始化，不要在代码中写死明文默认生产密码
- 后续如实现注册流程，请新增独立注册接口，不要改写登录语义
- 后续 WebSocket 鉴权阶段应复用当前 JWT 结构与签名配置
